# スパQ エラーカタログ

## 1. 文書情報

| 項目 | 内容 |
|---|---|
| 対象 | MVP |
| ステータス | Draft 1.0 |
| 更新日 | 2026-07-22 |
| 関連文書 | `docs/api-contract.md`、`docs/screen-flow-spec.md`、`docs/technical-requirements.md` |

本書は、画面とAPIが利用するエラーコード、利用者向けの固定文言、表示方法を定義する。APIの`error.message`には、原則として本書の「利用者向け文言」をそのまま設定する。

## 2. 共通表示ルール

- 利用者へは、内部エラー、HTTP status、DB名、Supabase、OneSignal、CAPTCHA事業者名を表示しない。
- 文言では「何が起きたか」「待ち列が維持されているか」「次の操作」を必ず示す。
- API失敗時は、現在の画面・入力内容を保持する。成功が確認できるまで画面遷移やローカルの管理トークン削除をしない。
- 状態変更ボタンは送信中に無効化し、二重送信を防ぐ。通信失敗後は再度有効化する。
- `retryable: true`は「再試行」ボタンを表示する。`false`は表示された遷移先または入力修正へ誘導する。
- 管理トークン不一致とエントリー不存在は、他人の待ち列の存在を推測できない同じ表示文言にする。
- Web Pushの失敗は待ち列を失効させない。待機画面の表示を正とする。
- 現地の空き、実際の並び順、施設ルールとアプリ表示が異なる場合は、常に現地を優先する。

### 2.1 表示コンポーネント

| 表示種別 | 用途 |
|---|---|
| フィールド直下 | ニックネーム・時間の入力不正。入力欄へフォーカスを戻す。 |
| 画面内バナー | ネットワーク、Realtime、Pushなど、画面を離れずに案内できるもの。 |
| 確認モーダル | 満車確認、退出、完了、延長など、利用者の判断が必要なもの。 |
| 結果画面 | 呼び出し失効・自動完了・自主退出など、待ち列が終了したもの。 |

## 3. カウントダウンと通常状態の扱い

カウントダウンの表示更新自体はエラーではない。`estimatedStartAt`、`callExpiresAt`、`expectedFinishAt`を基準に、端末内で表示だけを更新する。待ち列の計算結果が変わったときだけRealtimeまたはPollingで最新値へ同期する。

| 事象 | 表示・処理 |
|---|---|
| 待ち時間が再計算された | エラーやPushは出さず、順位・前方人数・「あと約○分」を静かに更新する。 |
| 待ち時間が0になったが、まだ`called`ではない | 「順番を確認しています…」と表示する。充電開始ボタンは出さない。サーバーの`called`状態を受信後にS-10へ遷移する。 |
| 呼び出しの5分が0になった | 充電開始ボタンを無効化する。APIが`CALL_EXPIRED`を返した場合、または端末内タイマー到達時にS-16を表示する。Cronは後続の繰り上げを行う。 |
| 終了予定の3分前になった | S-13の延長・終了確認モーダルを出す。これは正常な通知でありエラーではない。 |
| 終了予定の5分後まで無応答 | S-15で自動完了結果を表示する。後続がいる場合だけ、移動を促す強調文言を追加する。 |

画面を閉じていたために端末内タイマーが動かなかった場合、DB行はすでに削除されている。再表示時に`ENTRY_NOT_FOUND`となっても、自動完了・退出・失効のどれかをサーバーから推測して表示しない。

## 4. エラーコードと利用者向け文言

### 4.1 通信・同期

| Code | 発生元 / HTTP | 利用者向け文言 | 表示・操作 | 再試行 |
|---|---|---|---|---:|
| `NETWORK_OFFLINE` | ブラウザがオフライン | インターネットに接続できません。接続を確認してから、もう一度お試しください。待ち列の状態は接続後に確認できます。 | 画面内バナー。「再試行」 | Yes |
| `REQUEST_TIMEOUT` | API応答待ち時間超過 | 通信に時間がかかっています。待ち列への参加や操作が完了している可能性があるため、まず最新の状態を確認してください。 | バナー。「最新の状態を確認」 | Yes |
| `SERVER_TEMPORARY_ERROR` | `503` | 一時的に処理できませんでした。少し待ってから、もう一度お試しください。 | バナー。「再試行」 | Yes |
| `CONCURRENT_UPDATE` | `409` | 待ち列が更新されました。最新の内容を表示します。 | バナー。本人状態を自動再取得し、同じ操作は自動再送しない。 | No |
| `INVALID_QUEUE_STATE` | `409` | 待ち列の状態が更新されています。最新の内容を確認してください。 | バナー。「最新の状態を確認」 | Yes |
| `REALTIME_DISCONNECTED` | Realtime切断 | 最新情報への接続が不安定です。自動で再接続しています。 | 控えめなバナー。10〜15秒のPollingへ切替。 | 自動 |

### 4.2 施設検索・待ち列参加

| Code | 発生元 / HTTP | 利用者向け文言 | 表示・操作 | 再試行 |
|---|---|---|---|---:|
| `SEARCH_NO_RESULTS` | ブラウザ内検索 | 条件に一致する施設が見つかりませんでした。施設名、住所、都道府県を変えて検索してください。 | 検索結果の空状態 | No |
| `FACILITY_NOT_FOUND` | `404` | 施設情報が見つかりませんでした。もう一度検索してください。 | バナー。「施設を検索」 | No |
| `FACILITY_QUEUE_DISABLED` | `409` | この施設では現在、待ち列を受け付けていません。現地状況を優先してください。 | S-03/S-04のバナー | No |
| `FULL_CONFIRMATION_REQUIRED` | `409` | アプリ上の待ちはありません。現地が満車であることを確認してから、待ち列を開始してください。 | S-06を表示。「満車を確認して参加」 / 「戻る」 | No |
| `TERMS_NOT_ACCEPTED` | `400` またはクライアント入力検証 | 利用規約およびプライバシーポリシーへの同意が必要です。 | 同意チェックボックスの直下 | No |
| `TERMS_VERSION_OUTDATED` | `409` | 利用規約またはプライバシーポリシーが更新されています。内容を確認して同意してください。 | 同意欄と最新文書を表示 | No |
| `NICKNAME_INVALID` | `400` | ニックネームを1〜30文字で入力してください。 | ニックネーム欄の直下 | No |
| `ALREADY_JOINED` | クライアントの既存エントリー検出 | すでにこの待ち列へ参加しています。現在の待機画面を表示します。 | バナー。「待機画面を開く」 | No |
| `CAPTCHA_FAILED` | `400`、CAPTCHA有効時のみ | 操作を確認できませんでした。もう一度お試しください。 | CAPTCHA付近。「もう一度確認する」 | Yes |
| `RATE_LIMITED` | `429` | 操作が集中しています。しばらくしてから、もう一度お試しください。 | バナー。`Retry-After`があれば残り時間を表示。 | Yes |
| `IDEMPOTENCY_KEY_REUSED` | `409` | 同じ操作がすでに処理されています。最新の状態を確認してください。 | 最新状態を再取得。異なる入力で同じキーを再利用しない。 | No |

`FULL_CONFIRMATION_REQUIRED`は失敗画面ではない。待ち時間0分の施設で、利用者が現地満車を確認して待ち列を新規開始するための確認導線である。参加APIはロック後に再確認し、他の利用者が先に参加していた場合は通常参加として成功させる。

### 4.3 待ち列の所有・受付期限

| Code | 発生元 / HTTP | 利用者向け文言 | 表示・操作 | 再試行 |
|---|---|---|---|---:|
| `ENTRY_NOT_FOUND` | `404` | この待ち列の情報を確認できません。利用が終了したか、ブラウザのデータが変更された可能性があります。 | S-18。「施設を検索」 | No |
| `MANAGEMENT_TOKEN_INVALID` | `403` | この待ち列の情報を確認できません。利用が終了したか、ブラウザのデータが変更された可能性があります。 | S-18。「施設を検索」 | No |
| `START_NOT_AVAILABLE` | `409` | まだ充電開始を受け付けられません。順番の状態を確認してください。 | S-08〜S-10のバナー。「最新の状態を確認」 | Yes |
| `SKIP_START_NOT_AVAILABLE` | `409` | 現地の空きを待ち列へ反映できませんでした。周りの並びを確認して、最新の状態を確認してください。 | S-08/S-09のバナー。「最新の状態を確認」 | Yes |
| `CALL_EXPIRED` | `409` | 充電開始の受付時間を過ぎたため、待ち列を終了しました。必要な場合は現地状況を確認して、もう一度参加してください。 | S-16。「施設詳細へ」 | No |

### 4.4 充電時間・完了・延長

| Code | 発生元 / HTTP | 利用者向け文言 | 表示・操作 | 再試行 |
|---|---|---|---|---:|
| `DURATION_REQUIRED` | クライアント入力検証 | 予定充電時間を入力してください。 | S-11/S-14の入力欄直下 | No |
| `DURATION_NOT_INTEGER` | `400` またはクライアント入力検証 | 充電時間は1分単位の整数で入力してください。 | S-11/S-14の入力欄直下 | No |
| `DURATION_OUT_OF_RANGE` | `400` またはクライアント入力検証 | 充電時間は5〜120分で入力してください。 | S-11/S-14の入力欄直下 | No |
| `DURATION_ALREADY_CONFIRMED` | `409` | 最初の充電時間はすでに確定しています。追加する場合は、終了前の確認で延長してください。 | S-12へ戻す。「終了予定を確認」 | No |
| `EXTENSION_UNAVAILABLE` | `409` | 延長できる時間ではありません。現在の充電状態を確認してください。 | S-12/S-13のバナー。「最新の状態を確認」 | Yes |
| `COMPLETE_NOT_AVAILABLE` | `409` | 充電完了の処理を確認できません。現在の状態を確認してください。 | バナー。「最新の状態を確認」 | Yes |

|---|---|---|---|---:|
| `UNEXPECTED_ERROR` | 想定外の`5xx` | 処理を完了できませんでした。時間をおいて、もう一度お試しください。 | バナー。「再試行」 | Yes |

## 5. 終了結果の固定文言

これらはAPIエラーではなく、S-15〜S-17で表示する結果である。成功レスポンスまたは端末内カウントダウンの到達時だけ表示する。

| 結果 | 見出し | 本文 | 主操作 |
|---|---|---|---|
| 利用者が完了報告 | 充電が完了しました | ご利用ありがとうございました。次に使うときは、ニックネームを入力して待ち列へ参加してください。 | 「施設を探す」 |
| 無応答による自動完了・後続なし | 待ち列を終了しました | 充電終了の確認が取れなかったため、待ち列を終了しました。 | 「施設を探す」 |
| 無応答による自動完了・後続あり | 待ち列を終了しました | 後ろに待っている方がいます。充電が終わっている場合は、安全を確認して移動しましょう。 | 「施設を探す」 |
| 呼び出し失効 | 受付時間が終了しました | 5分以内に充電開始の報告がなかったため、待ち列を終了しました。 | 「施設詳細へ」 |
| 自主退出 | 待ち列から退出しました | 次に利用する場合は、ニックネームを入力してもう一度参加してください。 | 「施設を探す」 |

## 6. 実装ルール

1. Route Handlerは本書にない利用者向け`code`を返さない。内部例外は`UNEXPECTED_ERROR`または`SERVER_TEMPORARY_ERROR`へ変換する。`PUSH_SEND_FAILED`だけはCronの運用ログ用であり、ブラウザへ返さない。
2. `400`、`403`、`404`、`409`、`429`、`503`のいずれでも、レスポンスは`{ error: { code, message, retryable } }`を返す。
3. `429`は可能な場合に`Retry-After`ヘッダーを返す。画面はその時間が過ぎるまで同一操作ボタンを無効化する。
4. `REQUEST_TIMEOUT`の後に状態変更APIを自動再送しない。`GET /api/queue/me`で最新状態を確認してから利用者が再操作する。
5. `CALL_EXPIRED`と無応答による自動完了は、サーバーが後続の繰り上げを終えてから有効エントリーを削除する。履歴テーブルは作らない。
6. エラー監視へ送る情報は`code`、HTTP status、リクエストID、施設IDのハッシュ、時刻に限定する。ニックネーム、管理トークン、Push Subscription IDは送らない。
